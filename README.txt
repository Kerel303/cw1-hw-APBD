1. Kiedy Git wykona fast-forward, a kiedy powstaje merge commit?
Fast-forward wykona wtedy gdy nie ma nowych commitów, czyli po prostu git przesunie wskaźnik, natomiast merge commit polega na scaleniu
dwóch różnych (zmienionych) gałęzi
2. Czym w praktyce różni się merge od rebase?
Rebase nie zachowuje historii commitów z danego brancha
3. W jaki sposób został rozwiązany konflikt w Twoim repozytorium?
